# 代币注册表程序

## 新项目不推荐使用

代币注册表程序仍在主网上运行并继续服务现有集成。然而，它**不推荐用于新项目**。更新的代币标准目前正在积极的社区讨论中，预计将取代代币注册表：

- [ARC-20](https://github.com/ProvableHQ/ARCs/discussions/124) — Aleo 上同质化代币的提议标准

- [ARC-22](https://github.com/ProvableHQ/ARCs/discussions/125) — Aleo 上合规/受监管同质化代币的提议标准，扩展了 ARC-20

新项目应关注这些讨论，并在标准最终确定后考虑基于即将推出的标准进行构建。

## 概述

代币注册表程序是一个用于在 Aleo 上发行和管理代币的单例程序。与其为每个代币部署单独的程序，所有代币都向这个中央注册表注册，由注册表管理它们的余额。DeFi 程序仅依赖注册表，因此可以添加新代币而无需重新部署现有的 DeFi 程序。作为次要好处，注册表内的私密转账隐藏了正在转移的具体代币，提高了匿名集。

这种设计早于[动态调度（ARC-0009）](https://github.com/ProvableHQ/ARCs/tree/master/arc-0009)。随着动态调度的最终确定，程序可以在运行时调用其他程序而无需编译时导入，实现了 [ARC-20](https://github.com/ProvableHQ/ARCs/discussions/124) 中描述的每程序代币模型。代币注册表仍在主网上运行并继续服务现有集成 —— 有关新项目的指导，请参阅上面的警告。

该标准源自社区讨论和 [ARC-21 提案](https://vote.aleo.org/p/21) 的批准。

原始源代码可以在[这里](https://github.com/demox-labs/aleo-standard-programs/blob/main/token_registry/src/main.leo)找到。

## 如何使用代币注册表程序

调用 `register_token`，传入唯一的代币 ID、名称、符号、小数位数和最大供应量。将 `external_authorization_required` 设置为 `true` 需要在代币可以被花费之前获得 `external_authorization_party` 的批准，该方使用 `prehook_public` 或 `prehook_private` 在特定的过期区块高度为特定所有者解锁余额。管理员稍后可以通过 `update_token_management` 更新 `external_authorization_party`。

注册后，代币可以公开铸造（`mint_public`）或私密铸造（`mint_private），并可以公开销毁（`burn_public`）或私密销毁（`burn_private`）。如果调用者不是管理员，铸造和销毁需要 `MINTER_ROLE`/`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

所有者可以公开转账代币（`transfer_public`）或私密转账（`transfer_private`），并可以使用 `transfer_public_to_private` 和 `transfer_private_to_public` 在公共和私有余额之间转换。

## 数据结构

### Token 记录

```
record Token {

 owner: address,

 amount: u128,

 token_id: field,

 external_authorization_required: bool,

 authorized_until: u32           // 授权过期的区块高度

}
```

### TokenMetadata 结构体

```
struct TokenMetadata {

 token_id: field,

 name: u128,                          // ASCII 文本作为 u128 位串

 symbol: u128,                        // ASCII 文本作为 u128 位串

 decimals: u8,

 supply: u128,

 max_supply: u128,

 admin: address,

 external_authorization_required: bool,

 external_authorization_party: address

}
```

### TokenOwner 结构体

```
struct TokenOwner {

 account: address,

 token_id: field

}
```

### Balance 结构体

```
struct Balance {

 token_id: field,

 account: address,

 balance: u128,

 authorized_until: u32

}
```

### Allowance 结构体

```
struct Allowance {

 account: address,

 spender: address,

 token_id: field

}
```

## 映射

| 映射 | 键 | 值 |
|---|---|---|
| `registered_tokens` | `field`（代币 ID） | `TokenMetadata` |

| `allowances` | `field`（代币 ID + 账户 + 支出者的哈希） | `Allowance` |
| `roles` | `field`（代币 ID + 账户的哈希） | `u8` |

## 功能

### `initialize()`

使用硬编码的元数据注册 ALEO 积分代币（名称/符号为 "credits"，6 位小数，最大供应量 10 千万亿）。参数是硬编码的，以防止抢先交易。将 `wrapped_credits.aleo` 设置为管理员，外部授权被禁用，因此元数据之后无法修改。

### `register_token()`

注册新代币。调用者成为管理员。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 唯一代币标识符 |
| `name` | `public u128` | 代币名称 |
| `symbol` | `public u128` | 代币符号 |
| `decimals` | `public u8` | 小数位数 |
| `max_supply` | `public u128` | 最大供应量 |
| `external_authorization_required` | `public bool` | 是否需要外部授权 |
| `external_authorization_party` | `public address` | 授权方地址 |

返回：`Future`

### `update_token_management()`

更新代币的管理员和外部授权方。只能由当前管理员调用。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `admin` | `public address` | 新管理员地址 |
| `external_authorization_party` | `public address` | 新授权方地址 |

返回：`Future`

### `set_role()` / `remove_role()`

`set_role` 为给定代币的账户分配角色（`MINTER_ROLE`、`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`）。`remove_role` 清除该角色。两者都要求调用者是代币管理员。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `account` | `public address` | 目标账户 |
| `role` | `public u8` | 角色值（仅限 `set_role`） |

返回：`Future`

### `mint_public()` / `mint_private()`

向接收者铸造代币。需要管理员、`MINTER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `recipient` | `address`（`mint_private` 时为私有） | 接收者地址 |
| `amount` | `public u128` | 铸造金额 |
| `external_authorization_required` | `public bool` | （仅限 `mint_private`） |
| `authorized_until` | `public u32` | 授权过期区块高度 |

返回：`mint_public` → `Future`；`mint_private` → `Token`、`Future`

### `burn_public()` / `burn_private()`

销毁代币。需要管理员、`BURNER_ROLE` 或 `SUPPLY_MANAGER_ROLE`。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符（仅限 `burn_public`） |
| `owner` | `public address` | 代币所有者（仅限 `burn_public`） |
| `input_record` | `Token` | 代币记录（仅限 `burn_private`） |
| `amount` | `public u128` | 销毁金额 |

返回：`burn_public` → `Future`；`burn_private` → 剩余 `Token`、`Future`

### `prehook_public()` / `prehook_private()`

由 `external_authorization_party` 调用，以在指定的区块高度之前为所有者解锁指定数量的代币。

| 参数 | 类型 | 描述 |
|---|---|---|
| `owner` | `public address` | 代币所有者（仅限 `prehook_public`） |
| `input_record` | `Token` | 代币记录（仅限 `prehook_private`） |
| `amount` | `u128` | 授权金额 |
| `authorized_until` | `u32` | 授权过期区块高度 |

返回：`prehook_public` → `Future`；`prehook_private` → 未授权的 `Token`、已授权的 `Token`、`Future`

### `transfer_public()`

在公共余额之间转账代币。调用者是发送者（`self.caller`）。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `recipient` | `public address` | 接收者地址 |
| `amount` | `public u128` | 转账金额 |

返回：`Future`

### `transfer_public_as_signer()`

与 `transfer_public` 相同，但使用 `self.signer` 作为发送者，支持在任意程序调用链中使用。

参数和返回值与 `transfer_public` 相同。

### `approve_public()` / `unapprove_public()`

`approve_public` 授予支出者代表所有者转账代币的授权额度。`unapprove_public` 撤销或减少该授权额度。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `spender` | `public address` | 支出者地址 |
| `amount` | `public u128` | 授权或取消授权的金额 |

返回：`Future`

### `transfer_from_public()`

使用预先授权的授权额度，从所有者向接收者转账代币。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `owner` | `public address` | 代币所有者 |
| `recipient` | `public address` | 接收者地址 |
| `amount` | `public u128` | 转账金额 |

返回：`Future`

### `transfer_public_to_private()`

将公共余额转换为私有代币记录。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `recipient` | `address`（私有） | 接收者地址 |
| `amount` | `public u128` | 转换金额 |
| `external_authorization_required` | `public bool` | 是否需要授权 |

返回：`Token`、`Future`

### `transfer_from_public_to_private()`

代表所有者使用预先授权的授权额度，将公共余额转换为私有代币记录。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_id` | `public field` | 代币标识符 |
| `owner` | `public address` | 代币所有者 |
| `recipient` | `address`（私有） | 接收者地址 |
| `amount` | `public u128` | 转换金额 |
| `external_authorization_required` | `public bool` | 是否需要授权 |

返回：`Token`、`Future`

### `transfer_private()`

在私有记录之间转账代币。

| 参数 | 类型 | 描述 |
|---|---|---|
| `recipient` | `address`（私有） | 接收者地址 |
| `amount` | `u128` | 转账金额 |
| `input_record` | `Token` | 发送者的代币记录 |

返回：发送者剩余的 `Token`、接收者的新 `Token`、`Future`

### `transfer_private_to_public()`

将私有代币记录转换为公共余额。

| 参数 | 类型 | 描述 |
|---|---|---|
| `recipient` | `public address` | 接收者地址 |
| `amount` | `public u128` | 转换金额 |
| `input_record` | `Token` | 要消耗的代币记录 |

返回：剩余的 `Token`、`Future`

### `join()`

将两个相同代币的私有代币记录合并为一个。总金额不变。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token_1` | `private Token` | 第一个代币记录 |
| `token_2` | `private Token` | 第二个代币记录 |

返回：合并后的 `Token`

### `split()`

将一个私有代币记录拆分为两个。总金额不变。

| 参数 | 类型 | 描述 |
|---|---|---|
| `token` | `private Token` | 要拆分的代币记录 |
| `amount` | `private u128` | 拆分到第一个输出的金额 |

返回：拆分后的 `Token`、剩余的 `Token`
